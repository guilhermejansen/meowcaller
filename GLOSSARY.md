# Glossary

Acronyms and terms used across the datasheets and docs. One line each, grouped by
area. When a datasheet uses a term you do not know, it is here.

## Project

- **datasheet** — the per-module reference (`datasheets/<module>.md`): the source
  verbatim, the Go envelope, and suggestions. The thing you build a module from.
- **envelope** — the Go type and function signatures for a module, with no bodies.
- **reference** — the existing, validated implementation read verbatim into a
  datasheet. Never named in the Go code.
- **KAT** — Known-Answer Test: a fixed input plus its known-correct output (a
  "vector"), asserted byte/precision-exact. The proof a module is right.
- **vector** — a `*.json` file of KAT cases for a module (e.g. `toc_vectors.json`).
- **ADR** — a decision record written (on direction) to the spec repo when an
  implementation choice is settled.
- **spec / wacrg** — the abstract, implementation-independent protocol description
  (the "RFC"), kept in a separate documentation repo.

## Audio codec (the media layer)

- **MLow** — WhatsApp's default 1:1 call audio codec. Internally named **smpl**
  (`smpl_audio_codec`).
- **CELP** — Code-Excited Linear Prediction: the speech-codec family MLow belongs
  to (linear-prediction filter + an excitation signal).
- **TOC** — Table-of-Contents byte: the first byte of a frame, encoding how to
  decode it (and whether it is a standard-Opus frame to route elsewhere).
- **PCM** — Pulse-Code Modulation: raw audio samples (the decoder's output).
- **LPC** — Linear Predictive Coding: the short-term vocal-tract filter.
- **LSF / NLSF** — Line Spectral Frequencies (Normalized): a stable, quantizable
  representation of the LPC filter. (**LSP**, Line Spectral Pairs, is the same idea.)
- **LTP** — Long-Term Prediction: pitch (periodicity) prediction. Uses an
  **ACB** (Adaptive Codebook).
- **PVQ** — Pyramid Vector Quantization: how the excitation/innovation pulses are
  coded.
- **VQ / RD** — Vector Quantization / Rate-Distortion (the LSF quantizer search).
- **range coder / entropy coder** — the arithmetic coder that packs all symbols
  into the bitstream. MLow uses the unmodified Opus/CELT coder, whose functions are
  **ec_dec** / **ec_enc**; its tables are **(i)CDF** ((inverse) cumulative
  distribution functions).
- **VAD** — Voice Activity Detection: is this frame speech or not.
- **SID** — Silence Insertion Descriptor: a tiny "comfort noise" frame sent during
  silence under **DTX** (Discontinuous Transmission); regenerated by **CNG**
  (Comfort Noise Generation).
- **RED** — Redundant Encoding (RFC 2198): carrying older frames alongside the
  current one for loss resilience. A form of **FEC** (Forward Error Correction).
- **ROM (heap ROM)** — the large block of constant tables the codec reads from
  (codebooks, CDFs, contour tables).
- **MDCT** — Modified Discrete Cosine Transform. Used by the *standard-Opus*
  fallback path, **not** by MLow. Mentioned only to disambiguate.
- **Opus / CELT / SILK** — the standard codec and its two halves; MLow forks/reuses
  parts of this lineage.

## Keying and crypto

- **callKey** — the per-call media secret, delivered end-to-end via Signal. The
  root of every media key.
- **HKDF / KDF** — (HMAC-based) Key Derivation Function: expands a secret into keys.
- **SRTP / SRTCP** — Secure RTP / RTCP: encrypted+authenticated media / control.
- **E2E** — End-to-End: encryption only the two callers can read.
- **HBH** — Hop-by-Hop: a second SRTP layer keyed by the relay, so the relay can
  route but the inner E2E stays sealed.
- **AES-CM** — AES Counter Mode (the SRTP cipher and the RFC 3711 key-derivation
  PRF). **AES-GCM** — AES Galois/Counter Mode, authenticated (used by SFrame).
- **HMAC** — keyed hash used for authentication tags.
- **SFrame** — Secure Frames: a per-frame media E2EE layer (AES-128-GCM), separate
  from SRTP.
- **WARP** — WhatsApp's media transport framing over the relay, including its
  **MI** (MESSAGE-INTEGRITY) authentication tag.
- **ROC** — Roll-Over Counter: the high bits of the 48-bit packet index (SRTP/WARP).
- **KID** — Key ID: which key an SFrame frame used.
- **LID** — the participant's linked-device identifier (the `@lid` address); used
  as the HKDF `info` for per-participant keys.

## Transport and signaling

- **RTP / RTCP** — Real-time Transport Protocol and its Control Protocol.
- **SSRC** — Synchronization Source: the RTP stream identifier (here, derived, not
  random).
- **STUN** — the NAT-traversal message format; the relay speaks a WhatsApp dialect
  of it (custom message types and attributes).
- **ICE** — Interactive Connectivity Establishment: picking a working path/relay.
- **DTLS** — Datagram TLS. **SCTP** — the data-channel transport carried over DTLS.
- **relay** — the WhatsApp media server that forwards (encrypted) media when a
  direct path is unavailable.
- **stanza** — a signaling message: a WhatsApp binary (WABinary) XML-like node. The
  call-control stanzas are `<offer>`, `<preaccept>`, `<accept>`, `<transport>`,
  `<relaylatency>`, `<heartbeat>`, `<terminate>`, `<mute_v2>`, `<reject>`.
- **JID** — a WhatsApp address (account or device).
- **te / rte** — transport-endpoint blobs (relay candidates) carried inside stanzas.
- **keygen** — the key-generation scheme version (e.g. `encopt keygen="2"`).
